1、汇编：TASM

1-1、命令格式

TASM [options] sources [,object] [,listing] [,xref]

方括号内的选项都是可选项，注意复数形式代表可以有一个或多个选项。

1-2、选项解释

  • options：参数（≥1个）
  • sources：待汇编的源文件名，可以一次汇编多个源文件。
  • object：指定生成的目标文件名，如果缺省，则生成文件名为源文件名（大写）
  • listing：指定生成的列表文件名，如果缺省，则不生成列表文件
  • xref：指定生成的交叉引用文件名，如果缺省，则不生成交叉引用文件

1-3、帮助

使用 “tasm/?”或“tasm”命令可以得到TASM汇编软件用法的详细信息。

图1.1 TASM用法

1-4、部分参数解释

  • 对大小写是否敏感（win32中必须使用/ml） 
    • <UL>
<LI>/ml：全部敏感</LI>
<LI>/mx：全局变量大小写敏感</LI>
<LI>/mu：不敏感</LI>
</UL>
  • /m#：允许第#次编译扫描时可以向前引用，一般使用/m2
  • /i 路径：设定include文字所在的路径
  • list文件的格式 
    • <UL>
<LI>/l：正常格式</LI>
<LI>/la：扩充格式</LI>
</UL>
  • 符号调试信息的控制 
    • <UL>
<LI>/zi：全部</LI>
<LI>/zd：仅有行号</LI>
<LI>/zn：不产生调试信息</LI>
</UL>

2、编译：TCC

2-1、命令格式

TCC [options] files

方括号内的选项都是可选项，注意复数形式代表可以有一个或多个选项。

2-2、选项解释

  • options：TCC用到的编译选项
  • files：待编译的C源文件

2-3、帮助

使用 “tcc”命令可以得到TCC编译软件用法的详细信息。

图2.1 TCC用法

2-4、部分参数解释

  • -mt：选择Tiny Model
  • -c：只编译，仅生成obj文件

    • 3、链接：TLINK

    3-1、命令格式

    TLINK [option] objfiles [,exefile] [,mapfile] [,libfiles] [,deffile] [,resfiles]

    方括号内的选项都是可选项，注意复数形式代表可以有一个或多个选项。

    3-2、选项解释

    • option：TLINK用的到链接选项（1个），可以使用默认的选项。
    • objfiles：待链接的目标文件，可以有1个或多个。当多于一个时，需要用'+'连接起来。要注意其前后顺序。
    • exefile：链接后可执行程序的文件名，缺省时以第一个目标文件名为可执行程序名。
    • mapfile：生成的影像文件名，可省略。通过分析映像文件可以分析程序的性能。
    • libfiles：链接时用到的库文件，不需要库文件时可以省略该选项。
    • deffile：程序用到的定义文件
    • resfiles：程序用到的资源文件

    3-3、帮助

    使用 “tlink/?”或“tlink”命令可以得到TLINK链接软件用法的详细信息。

    图3.1 TLINK用法

    3-4、部分参数解释

    • /t：生成.COM文件。 
      • <UL>
<LI>需要注意的是.COM文件是一种不分段的程序结构，所有的代码、数据和堆栈都在一个物理段内。故而当汇编文件定义了多个段时，链接时将提示致命错误——存在可浮动装配的段，链接失败。</LI>
</UL>

    • 追踪方法 
      • <UL>
<LI>单步执行：每次执行一条指令，观察指令执行后寄存器、内存单元内容的变化，分析程序功能。</LI>
    <UL>               
    <LI>F7：表示Trace，按照CPU的实际执行过程，跟踪每一条指令的执行，是严格意义上的“单条指令”执行。</LI>
    <LI>F8：与F7有所不同，F8会将循环体执行一次，而F7会反复执行。</LI>
    <LI>上述不同仅仅发生在“代码调试”方式下，在“符号调试”中，此刻的F7和F8功能相仿。</LI>
    </UL>       
<LI>分段执行：每次执行一组指令，观察该段指令执行后寄存器、内存单元内容的变化，分析程序功能。
    <UL>               
    <LI>F2：在当前位置设置断点（该断点可多次使用）。</LI>
    <LI>F9：从当前指令开始执行到断点处。</LI>
    <LI>F4：从“当前指令”执行到光标条所在行停下（该“断点”只能使用一次）</LI>
    </UL>
</LI>
</UL>
    • 重新启动
      • 修改IP寄存器的值为初始地址使程序重新开始执行。
    • 查看地址内容： ALT+F10——goto，输入ds:offset，查看该位置处的内容
