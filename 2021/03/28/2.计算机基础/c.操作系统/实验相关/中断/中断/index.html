1、了解中断

1-1、定义

  中断是现代微型计算机系统广泛采用的资源共享技术。

  所谓中断是指CPU在正常执行程序的过程中,由于某个外部或内部事件的作用,强迫CPU停止当前正在执行的程序,转去为该事件服务(称为中断服务),待服务结東后,又能自动返回到被中断的程序中继续执行

1-2、中断的功能

  1. 实现CPU与外部设备的速度匹配与并行工作。
  2. 实现实时信息监测和控制。
  3. 实现故障检测和自动处理。

1-4、中断的种类

  • 内部中断：CPU在处理某些特殊事件而引起或通过自己内部逻辑去调用的中断 
    • <UL>
<LI>软中断：中断指令调用的中断</LI>
<LI>异常中断：由于CPU的某些错误引起的中断，如除法出错中断、溢出中断</LI>
<LI>陷阱中断：为调试程序设置的中断</LI>
</UL>
  • 外部中断：由于外部设备要求数据输入或输岀操作时请求CPU为之服务的中断 
    • <UL>
<LI>可屏蔽中断：<FONT COLOR=RED>外设</FONT>完成数据准备，请求进行数据传输引起的中断</LI>
<LI>不可屏蔽中断（NMI）：通常代表CPU外部的故障，例如存储器校验错</LI>
</UL>

1-5、中断优先级与嵌套

软件中断（内部中断） > 不可屏蔽中断（NMI） > 可屏蔽中断（INTR） > 单步中断

5、端口

5-1、什么是端口

  接口电路内部有一组寄存器，用于暂存CPU与外部设备之间传递的信息。为了使CPU能对这些寄存器进行读写，这些寄存器都分配到一个专用的地址，CPU可以用输入输出指令对这些寄存器进行读写，这种分配了地址的寄存器称为端口。 ## 5-2、端口分类
  • 数据输入端口：存放输入设备输入的数据信息，如键盘输入的字符代码
  • 数据输出端口：存放送往输出设备，准备输出的数据信息，如送往打印机的字符代码
  • 状态端口（输入）：存放外部设备/接口的状态信号，供CPU读取
  • 控制端口（输出）：存放CPU发往外部设备或接口的命令或控制信息

5-3、端口编址方式

###5-3-1、I/O端口与内存统一编址   从内存地址空间里划分出一部分，供I/O端口使用，CPU对端口的操作就如同使用内存单元一样。 ###5-3-2、I/O端口与内存独立编址   为I/O端口另外设置一套地址，端口不占用内存的地址空间，CPU对端口与对内存单元使用不同的操作指令。80x86微机采用该种方式，用16位二进制表示一个端口的地址，最多可以有65536个不同的端口。 ## 5-4、8086的输入输出指令   IN和OUT指令的共同特点是：必须使用累加器（ACC）AL或AX进行数据的传送。 ### 5-4-1、端口输入指令IN
  • 指令格式： 
    • <UL>
<LI>IN ACC,PORT</LI>
<LI>IN ACC,DX</LI>
</UL>
  • 操作： 
    • <UL>
<LI>AL/AX <- (PORT)</LI>
<LI>AL/AX <- (DX)</LI>
</UL>
  • 功能：把指定端口中的数据读入AL/AX中。
